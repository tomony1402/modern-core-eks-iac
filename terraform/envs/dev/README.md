# dev 環境について

本ディレクトリは、Terraform によるインフラ構成を
検証・試行するための開発環境（dev）を想定している。

## 目的

- Terraform の構成検証
- module 設計の試行
- 破壊的変更を含む apply / destroy の実施

## prod 環境との違い

- dev 環境は破棄・再作成を前提とする
- 本番相当の可用性や冗長性は考慮しない
- 本番環境（prod）に適用する前の確認用途とする

## 運用方針

- `terraform apply` は本ディレクトリ配下で実行する
- 本ディレクトリの state は dev 専用とする
- dev 環境はコストを最小限に抑える構成とする

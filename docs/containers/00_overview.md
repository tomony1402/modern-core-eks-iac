# コンテナ概要（Kubernetes 学習の前提）

## このドキュメントの目的
- Kubernetes を学ぶ前に、コンテナの基本的な役割を理解する
- 「Kubernetes が何をしていないか」を明確にする
- コンテナと Kubernetes の責任範囲を切り分ける

---

## なぜ Kubernetes の前にコンテナを学ぶのか

Kubernetes は **コンテナを直接操作するツールではない**。

Kubernetes 自体は以下を行わない：

- コンテナイメージを作らない
- コンテナをローカルで起動しない
- Linux プロセスを直接管理しない

代わりに Kubernetes は：

- 「どのイメージを」
- 「いくつ」
- 「どの Node で」
- 「どの状態で」

動かしたいかを **宣言的に定義**する。

実際のコンテナ操作は、Node 上の  
**container runtime（例：containerd）** が担当する。

# コンテナ操作：Docker 基本コマンド

## このドキュメントの目的
- コンテナを「自分の手で操作」できるようになる
- Kubernetes が内部で行っていることを想像できるようにする
- コンテナ = Linux プロセスであることを体感する

---

## 基本コマンド一覧

| コマンド | 説明 | 補足 |
|--------|------|------|
| docker run | コンテナを作成して実行 | 作成＋起動を同時に行う |
| docker start | 停止中のコンテナを起動 | 既存コンテナのみ |
| docker stop | コンテナを停止 | 中のプロセスが終了 |
| docker rm | コンテナを削除 | 停止中のみ削除可 |
| docker pull | イメージを取得 | Docker Hub 等から |
| docker push | イメージを登録 | CI/CD で使用 |
| docker images | ローカルのイメージ一覧 | キャッシュ確認 |
| docker rmi | イメージを削除 | 使用中は不可 |
| docker ps | 稼働中コンテナ一覧 | `-a` で全表示 |
| docker build | イメージ作成 | Dockerfile を使用 |
| docker attach | 起動中コンテナに接続 | 標準入出力を共有 |
| docker exec | コンテナ内でコマンド実行 | 実務で多用 |

---

## docker run の代表的なオプション

| オプション | 意味 | 補足 |
|-----------|------|------|
| -it | 対話的に実行 | 標準入力 + TTY を割り当て |
| -d | バックグラウンド実行 | 常駐プロセスとして起動 |
| -e | 環境変数を渡す | 設定をイメージから分離 |
| -p | ポートフォワード | ホスト:コンテナ |
| -v | ボリュームマウント | bind mount |

※ docker run のオプションはローカル実行向けのものが多く、
Kubernetes では同等の機能を別リソースで実現する。
